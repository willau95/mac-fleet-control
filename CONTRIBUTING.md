# Contributing to Mac Fleet Control

Thanks for your interest! Here's how to contribute:

## Getting Started

1. **Fork** this repository
2. **Clone** your fork locally
3. **Create a branch** for your changes: `git checkout -b feature/my-change`
4. **Make your changes** and test them
5. **Commit** with a clear message: `git commit -m "feat: add xyz"`
6. **Push** to your fork: `git push origin feature/my-change`
7. **Open a Pull Request** against `main`

## Rules

- All changes must go through a **Pull Request** — direct pushes to `main` are blocked
- PRs require **at least 1 approval** before merging
- Keep commits clean and atomic
- Follow existing code style

## Commit Convention

```
feat: new feature
fix: bug fix
docs: documentation only
refactor: code restructuring
chore: maintenance tasks
```

## Security

- **Never** commit credentials, API keys, or private keys
- **Never** commit real IP addresses or hostnames
- See [SECURITY.md](SECURITY.md) for full security policy

## Questions?

Open an issue and we'll help you out.
